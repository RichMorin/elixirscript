ElixirScript

Translates a subset of Elixir into JavaScript.

Requirements

  • Elixir
  • Node/IO.js

Usage

Building from source (recommended)

  • Get source 
    git clone git@github.com:bryanjos/ex_to_js.git
  • Get Dependencies 
                  mix deps.get
npm install
  • Make distributable 
    MIX_ENV=prod mix ex2js.dist
  • uncompress dist/ex2js.tar.gz.
  • use ex2js executable found in the ex2js/bin folder
  • Optionally you can use the mix command, 
    mix ex2js.install
    which will install the distributable at /usr/local/ex2js

Using a release version

  • Download the latest release from releases
  • uncompress ex2js.tar.gz.
  • use ex2js executable found in the ex2js/bin folder

You can use the included mix task to convert Elixir to JavaScript

          
          $ ex2js -h
            usage: ex2js <input> [options]

            <input> path to elixir files or 
                    the elixir code string if the -ex flag is used

            options:

            -o  --output [path]   places output at the given path
            -t  --ast             shows only produced spider monkey ast
            -ex --elixir          read input as elixir code string
            -st --stdio           reads from stdio
                --lib             writes the standard lib js to standard out
            -h  --help            this message

For more information, please view the README and the issues on the repo. Currently the standard library isn't fully implemented and some of the language features aren't either.

NOTE: Since this targets ES6, using something like babeljs is needed to translate it to ES5 JavaScript in order to be used in the browser

Features

  • Interoperability with exisiting JavaScript libraries
  • Targets ES6
  • Limited differences from Elixir proper (ElixirScript is still in development and some features from Elixir are still missing or incomplete)

Differences from Elixir

Modules must be explicitly imported

Because modules are translated into ES6 modules, when using them in ElixirScript code, you must explicitly import them.

import will import either all functions from the module or a subset of functions defined (if the module supports it). 

        
          defmodule Intro do
              import Hello.World #translates to "import * as World from 'hello/world'"
              import US, only: [la: 1] #translates to "import la from 'us'"
          end

alias and require work the same. They both work as default import statements 

        
          defmodule Intro do
              alias Hello.World #translates to "import World from 'hello/world'"
              require US #translates to "import US from 'us'"
          end

import, alias, and require all except a from clause that allows the import path to be customized 

        
          defmodule Intro do
              alias Hello.World, from: "some/other/path" #translates to "import World from 'some/other/path'"
          end
